<objective>
Build the OCI pull client and download orchestrator -- the engine that fetches schema artifacts from OCI registries and extracts them to the local cache.

Purpose: This enables automatic schema download when load() statements reference OCI artifacts. The client handles the OCI Distribution API (manifest + blob fetch with token auth). The downloader orchestrates: cache check, credential resolution, pull, tar extraction, and deduplication of concurrent requests.

Output: Two modules (`src/oci/client.ts`, `src/oci/downloader.ts`) with unit tests, plus `nanotar` added as a production dependency.

  <behavior>
    - pullManifest fetches GET /v2/{repo}/manifests/{tag} with Accept: application/vnd.oci.image.manifest.v1+json
    - pullManifest handles 401 by parsing Www-Authenticate, exchanging for Bearer token, and retrying
    - pullManifest with credentials sends Basic auth to token endpoint
    - pullManifest without credentials sends anonymous request to token endpoint
    - pullBlob fetches GET /v2/{repo}/blobs/{digest} and returns Uint8Array
    - pullBlob reuses the Bearer token from the manifest request
    - pullArtifact combines manifest + blob fetch, returns tar data as Uint8Array

    - Error on non-200 blob response throws with status and digest info

    ```
    class OciClient {
      constructor(registryHost: string, repository: string, credentials?: DockerCredentials)
      async pullArtifact(tag: string): Promise<Uint8Array>
    }
    ```

    Internal methods:
    - `authenticatedFetch(url: string, accept: string): Promise<Response>` -- makes initial request. If 401, calls `parseWwwAuthenticate` on the `www-authenticate` header, fetches token from the `realm` URL (with Basic auth if credentials provided, anonymous otherwise), caches the Bearer token, retries original request with `Authorization: Bearer {token}`.
    - `getManifest(tag: string): Promise<OciManifest>` -- GET `/v2/{repo}/manifests/{tag}` with Accept `application/vnd.oci.image.manifest.v1+json`. Parse JSON response. Extract first layer's digest.
    - `getBlob(digest: string): Promise<Uint8Array>` -- GET `/v2/{repo}/blobs/{digest}`. Return `new Uint8Array(await response.arrayBuffer())`.
    - `pullArtifact(tag: string)` -- calls getManifest then getBlob, returns the Uint8Array tar data.

    Import `getDockerCredentials` and `parseWwwAuthenticate` from `./auth`.

    Use Node built-in `fetch()` (available in Node 18+ / VSCode 1.85+). No external HTTP library.

    - Pitfall 8: Use `new Uint8Array(await response.arrayBuffer())` for nanotar compatibility

  <files>src/oci/downloader.ts, src/oci/downloader.test.ts, package.json, package-lock.json</files>
  <behavior>
    - ensureArtifact skips download when cache directory already exists for the OCI ref
    - ensureArtifact pulls and extracts when cache directory does not exist
    - Tar entries are extracted to {cacheDir}/{artifactName}/{tag}/{entryPath}
    - Intermediate directories are created with recursive: true
    - Only file-type tar entries are extracted (directories, symlinks ignored)
    - Concurrent calls to ensureArtifact with the same OCI ref return the same promise (dedup)
    - Concurrent calls with different OCI refs run independently
    - In-flight promise is removed from the map after completion (success or failure)
    - Failed download does not leave a partially-written cache directory
    - nanotar parseTar is used for tar extraction
  </behavior>
  <action>
    First, install nanotar:
    ```bash
    npm install nanotar
    ```

    Create `src/oci/downloader.ts` exporting an `OciDownloader` class:

    ```
    class OciDownloader {
      constructor(cacheDir: string, defaultRegistry: string, log?: LogOutputChannel)
      async ensureArtifact(ociRef: string): Promise<string>  // returns artifact cache dir path
    }
    ```

    Internal logic:
    - `ensureArtifact(ociRef)`:
      1. Parse ociRef with `resolveOciRef(ociRef, this.defaultRegistry)` from `../load-parser`
      2. Compute artifact cache path: `{cacheDir}/{artifactName}/{tag}` where artifactName is the last segment of repository, tag is from resolved ref
      3. If `fs.existsSync(artifactCacheDir)` -- return early (cache hit, version is immutable per user decision)
      4. Check `this.inFlight` Map<string, Promise<string>>. If key exists, return existing promise (Pitfall 5: dedup concurrent downloads)
      5. Create and store promise in inFlight map. On completion (success or error), remove from map.
      6. Call `new OciClient(registryHost, repository, await getDockerCredentials(registryHost)).pullArtifact(tag)`
      7. Extract with `parseTar()` from nanotar. Write to temp dir first, then rename to final path (atomic cache population, prevents partial cache on failure).
      8. For each tar entry where `entry.type === 'file'` (or undefined -- nanotar uses undefined for regular files): create parent dirs with `fs.mkdirSync(dirname, { recursive: true })`, write `fs.writeFileSync(outPath, Buffer.from(entry.data!))`.
      9. Log download progress to the LogOutputChannel if provided.

    IMPORTANT about nanotar: `parseTar()` is for plain (uncompressed) tar. The OCI layer media type is `application/vnd.fn-starlark.layer.v1.tar` (plain tar, not gzipped). If the layer were gzipped, use `parseTarGzip()` instead. Since the user specified plain tar, use `parseTar()`.

    IMPORTANT about atomicity: Write extracted files to a temp directory (e.g., `{cacheDir}/{artifactName}/{tag}.tmp.{random}`), then rename to final path. If download/extraction fails, clean up the temp directory. This prevents other code from seeing a partially-populated cache.

    Create `src/oci/downloader.test.ts` with vitest tests. Mock `fs`, mock `OciClient` (vi.mock("./client")), mock `nanotar` (vi.mock("nanotar")). Test cache hit path. Test download + extract path. Test deduplication. Test cleanup on failure. This module has zero vscode dependency (log is optional and typed as any for testing).
