    - Returns empty diagnostics when no schema symbols are used without import
    - Diagnostics update on document open and document save
  </behavior>

    - Add `DiagnosticSeverity` enum: `{ Error: 0, Warning: 1, Information: 2, Hint: 3 }`
    - Add `Diagnostic` class: `constructor(range: Range, message: string, severity?: number)` with `range`, `message`, `severity` properties
    - Add `CodeAction` class: `constructor(title: string, kind?: unknown)` with `title`, `kind`, `edit`, `diagnostics` properties

    - Add `WorkspaceEdit` class with `insert(uri, position, text)` method (vi.fn)

    Create `src/diagnostics.ts` exporting:

    - `MissingImportDiagnosticProvider` class:
      ```
      class MissingImportDiagnosticProvider {
        constructor(schemaIndex: SchemaIndex, diagnosticCollection: vscode.DiagnosticCollection)
        updateDiagnostics(document: vscode.TextDocument): void
        provideCodeActions(document: vscode.TextDocument, range: vscode.Range, context: vscode.CodeActionContext): vscode.CodeAction[]
        dispose(): void
      }
      ```

    `updateDiagnostics(document)`:
    1. Parse load() statements from document text via `parseLoadStatements`
    2. Build set of imported symbols from the load statements (including star expansion)
    3. Union with `BUILTIN_NAMES`
    4. Scan document text for identifiers that match symbols in `schemaIndex.getAllSymbols()` but are NOT in the imported+builtin set
    5. For each unimported symbol, find its position in the document text and create a `vscode.Diagnostic` with severity `Hint`
    6. Set diagnostics on the collection

    Symbol usage detection: Use regex `/\b([A-Z]\w*)\s*\(/g` to find PascalCase function calls (schema constructors are always PascalCase). This avoids false positives on common variable names. Check if the matched name exists in `schemaIndex.getAllSymbols()`.

    `provideCodeActions(document, range, context)`:
    For each diagnostic in `context.diagnostics` that belongs to this provider, create a `CodeAction` with `kind: CodeActionKind.QuickFix` that inserts a load() statement. The action should use a `WorkspaceEdit` to insert the load statement. Position: after the last existing load() statement in the file (find with regex), or at line 0 if no existing loads.

    To determine which file to load from, use a reverse lookup on the SchemaIndex: find which cached file contains the symbol. Store this mapping during index build.

    Create `src/diagnostics.test.ts` with vitest tests. Mock vscode via the alias. Test diagnostic generation for unimported symbols. Test that builtins are not flagged. Test quick-fix code action generation. Test empty case (no unimported symbols). Test that unknown symbols (not in index) are not flagged.

    2. **Register document open/save handlers** (only when schemas enabled):
       ```typescript
       async function handleDocumentForSchemas(document: vscode.TextDocument) {
         if (document.languageId !== "starlark" || !downloader || !schemaIndex) return;
         const text = document.getText();
         const loads = parseLoadStatements(text);
         // Trigger downloads for uncached OCI refs
         for (const load of loads) {
           downloader.ensureArtifact(load.ociRef).then(() => {
             schemaIndex!.rebuild(getSchemaCachePath(context));
             updateDocumentImports(document.uri.toString(), text, schemaIndex!);
           }).catch(() => { /* logged by downloader */ });
         }
         // Update document imports cache for middleware filtering
         updateDocumentImports(document.uri.toString(), text, schemaIndex);
         // Update diagnostics
         diagnosticProvider?.updateDiagnostics(document);
       }

       context.subscriptions.push(
         vscode.workspace.onDidOpenTextDocument(handleDocumentForSchemas),
         vscode.workspace.onDidSaveTextDocument(handleDocumentForSchemas),
         vscode.workspace.onDidCloseTextDocument((doc) => {
           clearDocumentImports(doc.uri.toString());
         }),
       );
       ```

       context.subscriptions.push(

           await fs.promises.rm(cacheDir, { recursive: true, force: true });
           await fs.promises.mkdir(cacheDir, { recursive: true });
           schemaIndex?.rebuild(cacheDir);
           if (client?.isRunning()) await client.restart();
