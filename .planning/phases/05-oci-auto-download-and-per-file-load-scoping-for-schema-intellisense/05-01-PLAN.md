    - parseLoadStatements with multiple symbols: load("...", "Deployment", "StatefulSet") returns symbols: ["Deployment", "StatefulSet"]
    - parseLoadStatements with star import: load("...", "*") returns symbols: ["*"]
    - parseLoadStatements with full OCI URI: load("ghcr.io/org/schemas:v2.0/path/file.star", "Foo") returns ociRef: "ghcr.io/org/schemas:v2.0"
    - parseLoadStatements ignores non-OCI paths: load("//lib:utils.star", "helper") returns empty array
    - parseLoadStatements ignores paths without .star extension: load("schemas-k8s:v1.31/apps/v1", "Deployment") returns empty
    - parseLoadStatements ignores incomplete paths: load("schemas-k8s", "Deployment") returns empty (no colon tag with slash after)
    - parseLoadStatements handles multiple load() statements in one file
    - parseLoadStatements ignores load() with no symbols
    - isOciLoadPath("schemas-k8s:v1.31/apps/v1.star") returns true
    - isOciLoadPath("//lib:utils.star") returns false (colon is part of Bazel label, no slash after tag)
    - isOciLoadPath("schemas-k8s:v1.31") returns false (no .star extension, no slash after tag)
    - splitOciPath("schemas-k8s:v1.31/apps/v1.star") returns {ociRef: "schemas-k8s:v1.31", tarEntryPath: "apps/v1.star"}
    - splitOciPath("ghcr.io/org/schemas:v2.0/path/file.star") returns {ociRef: "ghcr.io/org/schemas:v2.0", tarEntryPath: "path/file.star"}
    - resolveOciRef("schemas-k8s:v1.31", "ghcr.io/wompipomp") returns {registryHost: "ghcr.io", repository: "wompipomp/schemas-k8s", tag: "v1.31"}
    - resolveOciRef("ghcr.io/org/schemas:v2.0", "ghcr.io/wompipomp") returns {registryHost: "ghcr.io", repository: "org/schemas", tag: "v2.0"}
  </behavior>
  <action>
    Create `src/load-parser.ts` exporting:
    - `LoadStatement` interface: `{ ociRef: string; tarEntryPath: string; symbols: string[]; fullPath: string }`
    - `OciRefResolved` interface: `{ registryHost: string; repository: string; tag: string }`
    - `parseLoadStatements(text: string): LoadStatement[]` -- regex-based extraction of load() calls. Use `/load\(\s*"([^"]+)"((?:\s*,\s*"[^"]*")*)\s*\)/g` for the statement, then extract symbols from the comma-separated part. Skip non-OCI paths via `isOciLoadPath`. Skip entries with zero symbols.
    - `isOciLoadPath(path: string): boolean` -- validates: must have colon with content before it, must end with `.star`, must have a `/` after the colon-tag portion. This distinguishes OCI refs (`schemas-k8s:v1.31/apps/v1.star`) from Bazel labels (`//lib:utils.star`).
    - `splitOciPath(fullPath: string): { ociRef: string; tarEntryPath: string }` -- splits at first `/` after the colon. Everything before is the OCI ref, everything after is the tar entry path.
    - `resolveOciRef(ociRef: string, defaultRegistry: string): OciRefResolved` -- if ociRef contains a dot before the colon (indicates hostname), treat as full URI and split at first `/` for registryHost. Otherwise, treat as short path: split defaultRegistry at first `/` for registryHost, combine remainder with artifact name for repository.

    IMPORTANT: Reset regex lastIndex before each call since using `g` flag. Better: create regex inside the function to avoid shared state.

    Create `src/load-parser.test.ts` with vitest tests covering all behaviors above. This module has zero vscode dependency -- pure Node.js, no mock setup needed.
  </action>
  <verify>
    <automated>npx vitest run src/load-parser.test.ts</automated>
  </verify>
  <done>All load parser tests pass. parseLoadStatements correctly extracts OCI load statements with symbols, isOciLoadPath validates format, splitOciPath separates ref from entry path, resolveOciRef handles both short and full URI forms.</done>
